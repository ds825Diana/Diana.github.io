# Portfolio v2 — Setup Guide

## File Structure

```
portfolio/
├── index.html          # Main site (bilingual EN/ZH, reads content.json)
├── admin.html          # Content editor (runs locally or self-hosted)
├── content.json        # Generated by admin — site reads this on load
├── resume.pdf          # Drop your CV here
├── .github/
│   └── workflows/
│       └── deploy.yml  # Auto-deploy to GitHub Pages on push
└── README.md
```

---

## Step 1 — Edit your content

Open `admin.html` in a browser (double-click or `open admin.html`).

Fill in:
- **Profile & Meta** — your name, city, degree, social links, bio in both languages
- **Projects** — add each project with EN + ZH title/description, tech stack, year, URL
- **Skills** — edit categories and tags

Then go to **Export / Deploy** → **Download content.json**.

Place the downloaded `content.json` next to `index.html`.

---

## Step 2 — Push to GitHub

```bash
git init
git add .
git commit -m "initial: portfolio site v2"

# Name the repo YOUR_USERNAME.github.io for a clean URL
git remote add origin https://github.com/YOUR_USERNAME/YOUR_USERNAME.github.io
git branch -M main
git push -u origin main
```

---

## Step 3 — Enable GitHub Pages

1. GitHub repo → **Settings** → **Pages**
2. Source → **GitHub Actions**
3. Save — site goes live at `https://YOUR_USERNAME.github.io` in ~60s

## Option B — Supabase backend (dynamic, no redeploy needed)

With Supabase, you edit content in `admin.html` and hit "Save to Supabase" —
the live site updates instantly without pushing to GitHub.

### 1. Create a Supabase project

Go to [supabase.com](https://supabase.com) → New Project (free tier is fine).

### 2. Create the content table

In Supabase SQL Editor, run:

```sql
create table portfolio_content (
  id   integer primary key default 1,
  data jsonb   not null default '{}'
);

-- Allow public read (the portfolio site reads this)
create policy "public read" on portfolio_content
  for select using (true);

-- Allow authenticated write (your admin panel)
create policy "authenticated write" on portfolio_content
  for all using (auth.role() = 'authenticated');

alter table portfolio_content enable row level security;
```

### 3. Get your credentials

Project Settings → API:
- **Project URL**: `https://xxxx.supabase.co`
- **anon public key**: `eyJhbGciOiJIUzI1...`

### 4. Configure the site to read from Supabase

In `index.html`, find `loadContent()` and add above the `fetch('content.json')` line:

```js
const SB_URL = 'https://YOUR_PROJECT.supabase.co';
const SB_KEY = 'YOUR_ANON_KEY';

// Try Supabase first
try {
  const r = await fetch(`${SB_URL}/rest/v1/portfolio_content?id=eq.1`, {
    headers: { 'apikey': SB_KEY, 'Authorization': 'Bearer '+SB_KEY }
  });
  const rows = await r.json();
  if (rows.length) {
    const data = rows[0].data;
    if (data.skills)   renderSkills(data.skills);
    if (data.projects) renderProjects(data.projects);
    return;
  }
} catch {}
// fallback to content.json...
```

### 5. Save content from admin panel

Open `admin.html` → Export / Deploy → fill in Supabase URL + anon key → **Save to Supabase**.

## Updating content after launch

### Option A (file-based):
1. Open `admin.html` locally
2. Edit content → Export → Download `content.json`
3. `git add content.json && git commit -m "update content" && git push`

### Option B (Supabase):
1. Open `admin.html` locally
2. Edit content → Save to Supabase
3. Done — site updates live

---

## Bilingual system

The site uses `data-lang="en"` / `data-lang="zh"` attributes.
The `中文 / EN` toggle button in the nav switches the language and stores preference in `localStorage`.

All content entered in admin (projects, skills, bio) is stored with both `_en` and `_zh` fields.
